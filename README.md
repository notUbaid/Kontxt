## 🌟 What is Kontxt?

Kontxt is not a code editor. It is a **project brain and software-building curriculum** wrapped into an interactive platform. 

In an era where AI agents (like Cursor, Windsurf, Lovable, Claude Code and GitHub Copilot) can write the code for you, the bottleneck is no longer *how* to code, but *what* to code. Kontxt bridges this gap by teaching you how to think like a Software Architect and Product Builder. 

It guides you phase-by-phase through the software development lifecycle (SDLC) ensuring your AI tools have the exact **Context** they need to generate production-ready software without hallucinating or building the wrong thing.

## 🎯 What it helps with

- **Curing "Blank Canvas Syndrome":** Gives you a step-by-step roadmap from raw idea to launched product.
- **Architectural Decision Making:** Forces you to make key decisions (Tech Stack, DB Schema, UX Flows) *before* you prompt an AI to write code.
- **Project Memory:** Acts as a centralized knowledge base so your AI agents don't forget the core constraints of your app midway through development.
- **Validating Outputs:** Provides strict checklists to ensure what your AI built actually aligns with standard engineering best practices.

## 👥 Who is it for?

- **Non-Technical Founders ("Vibe Coders"):** People relying on AI tools to build their startups who need a structured framework to prevent spaghetti code.
- **Junior Developers:** Engineers looking to learn end-to-end system architecture and product management.
- **Hackathon Participants:** Teams that need to rapidly scope, design, and ship an MVP in 48 hours without getting bogged down in feature creep.
- **Solo Developers (Indie Hackers):** Builders who need an organized system to manage complex full-stack projects solo.

## ✨ Core Features

### 🔄 Multi-Mode Frameworks
Whether you are building a SaaS, a mobile app, or competing in a hackathon, Kontxt adapts its curriculum to your specific needs:
- **Production Mode:** Focuses on security, scalability, and maintainability.
- **Hackathon Mode:** Strips away the noise, focusing purely on speed, MVP delivery, and demo scripts.
- **Personal Mode:** Optimizes for learning and low-cost tool constraints.

### 📚 Interactive Phase Curriculum
A rigorous, 6-Phase progression system:
1. **Phase 0:** Discovery & Validation (PRDs, Personas, Competitor Analysis)
2. **Phase 1:** Product Design (Wireframes, Information Architecture, Design Systems)
3. **Phase 2:** System Architecture (Tech Stack, DB Schemas, Endpoints)
4. **Phase 3:** Development (Core loops, Auth, Integrations)
5. **Phase 4:** Production Readiness (Security, CI/CD, Monitoring)
6. **Phase 5/6:** Launch & Growth (SEO, Analytics, Retention)

### 🤖 High-Value AI Prompts
Every topic comes with carefully engineered, copy-pasteable AI prompts designed to be fed into Claude, Gemini, or Cursor to yield elite-level architectural advice and code scaffolding.

### 📝 Project Context Editor
A built-in markdown editor that saves your specific project decisions directly to the cloud via Supabase, ensuring you can pull your exact context from any device.

## 🛠 Tech Stack

Kontxt is built with modern web technologies:
- **Frontend:** React 19 + TypeScript
- **Build Tool:** Vite
- **Styling:** Tailwind CSS + Lucide Icons
- **Backend / Auth:** Supabase
- **State Management:** Zustand
- **Routing:** React Router DOM
- **Deployment:** Vercel
